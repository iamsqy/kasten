7.1 Customisable Variables ¶

Kasten is customisable using ‘customize’, via M-x customize-group RET kasten RET. Alternatively, one may use setq. For example, to change the variable ‘kasten-directory’ to ‘~/notes’ using setq, one may add the following code to their Emacs configuration file (such as ‘init.el’, preferably under ‘use-package’). 

(setq kasten-directory (expand-file-name "~/notes/"))

Default values in the manual may be evaluated, simplified or slightly modified. Please refer to the source code and search for ‘defcustom’.

Variable: kasten-directory ¶

Note directory.

Default value: ‘~/jrn/’.

Variable: kasten-file-extensions ¶

List of searchable file extensions for Kasten.

Default value: ‘("org" "txt" "tex")’.

Variable: kasten-default-extension ¶

Default extension for new notes.

Default value: ‘org’.

Variable: kasten-note-title-template ¶

Template for new notes.

Default value: ‘#+title:’.

Variable: kasten-index-hidden-files ¶

If non-nil, include dot files when indexing. May cause problem if backup files present in the directory.

Default value: ‘nil’.

Variable: kasten-search-function ¶

Function used for searching within the Kasten directory.

Default value: ‘consult-ripgrep’.

Variable: kasten-title-regexp ¶

Regexp to match the title of a note.

Default value: ‘"^#\\+[tT][iI][tT][lL][eE]: *\\(.*\\)$"’.

Variable: kasten-title-max-pos ¶

Max position of the title of a note. Depth for Kasten to match ‘kasten-title-regexp’.

Default value: ‘4096’.

Variable: kasten-category-regexp ¶

Regexp to match the category of a note.

Default value: ‘"^#\\+[cC][aA][tT][eE][gG][oO][rR][yY]: *\\(.*\\)$"’.

Variable: kasten-empty-category-placeholder ¶

If non-empty, display in category field for uncategorised notes. Leave empty to omit empty category for uncategorised notes.

Default value: ‘""’.

Variable: kasten-category-max-pos ¶

Max position of the category of a note. Depth for Kasten to match ‘kasten-category-regexp’.

Default value: ‘4096’.

Variable: kasten-title-category-split ¶

Split title and category in Kasten view.

Default value: ‘" --- "’.

Variable: kasten-tag-regexp ¶

Regexp to match a tag of a note.

Default value: ‘"#\\([[:alnum:]_-]+\\)"’.

Variable: kasten-tag-max-pos ¶

Max position of the title of a note. Depth for Kasten to match ‘kasten-tag-regexp’.

Default value: ‘65536’.

Variable: kasten-tag-symbol ¶

Leading symbol for tag.

Default value: ‘"#"’.

Variable: kasten-tag-first-char-regexp ¶

Regexp for the tag char to make ‘kasten-search-function’ understand.

If using ‘consult-ripgrep’, use ‘\x23’ instead of ‘#’.

Default value: ‘"\\x23"’.

Variable: kasten-id-symbol ¶

Leading symbol for ID.

Default value: ‘"§"’.

Variable: kasten-id-format ¶

Time format for generating ID. Use ‘%E’ for shortened title. See man page ‘strftime(3)’ for wildcards.

Default value: ‘"%y%m%d-%H%M-%E"’.

Variable: kasten-folder-timeformat ¶

Time format for folder. See man page ‘strftime(3)’ for wildcards.

Default value: ‘"%Y%m"’.

Variable: kasten-id-clash-time-inc ¶

Time interval to increment if the time-based ID clashes, in seconds. Set to 60 for 1 minute.

Default value: ‘60’.

Variable: kasten-id-regexp ¶

Regexp for an ID in a note.

Default value: ‘"§\\([0-9]\\{6\\}-[0-9]\\{4\\}\\)"’.

Variable: kasten-backlink-comment ¶

Insert before backlink ID when creating new note at point.

Default value: ‘#+backlink:’.

Variable: kasten-buffer-title ¶

Kasten mode buffer title.

Default value: ‘"Kasten"’.

Variable: kasten-minor-mode-lighter ¶

Function used for searching within the Kasten directory.

Default value: ‘Kt’.

Variable: kasten-auto-refresh ¶

If non-nil, automatically refresh Kasten buffer when files change.

Default value: ‘t’.
